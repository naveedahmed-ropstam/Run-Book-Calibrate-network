Platform-Specific Guidance

Confluence

  • Use a top-level space "CN Operations" with the page tree above as child pages.
  • Create templates for Process Page (17-section structure) and Incident PIR.
  • Use labels: order, inventory, integration, client-[name], edi.
  • Embed Mermaid via plugin or attach exported PNG diagrams.
  • Link Jira tickets to relevant runbook pages.

Notion

  • Use a database for Troubleshooting Matrix with filterable columns.
  • Process docs as templated pages with toggle headings per section.
  • Gallery view for Critical Screens with screenshots.
  • Sync block or embed link to this React handbook for interactive diagrams.

SharePoint

  • Document library per section with metadata: Owner, ReviewDate, System.
  • Use SharePoint search with managed properties for quick lookup.
  • Version history enabled on all runbooks.
  • Co-authoring for team updates; restrict client-profile edits to owners.

Google Docs / Drive

  • Folder structure mirrors page tree exactly.
  • One Google Sheet for Troubleshooting Matrix (filterable).
  • Use Google Sites or a linked React app for navigation shell.
  • Comment-based review workflow with assigned reviewers.

Documentation Ownership

Role Responsibility
Handbook Owner (Ops Manager) Overall structure, review cadence, approves major changes
Integration Doc Owner (L2 Lead) Integration guide, sync schedules, mapping tables
Warehouse Doc Owner (WH Manager) Pick/pack/ship, receiving, cycle count sections
EDI Doc Owner (Channel Manager) Rithum screens, EDI docs, retailer profiles
Client Profile Owners (Account Managers) Per-client SLA, routing, contacts
All Support Analysts Suggest updates from tickets; add matrix entries

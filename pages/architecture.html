ℹ️
The mental model

Think of three layers: Demand (Shopify + Retailers via Rithum) generates orders, the Brain (TechOMS) decides what happens, and the Hands (Logiwa + Carriers) physically fulfill. Inventory flows up from the hands; orders flow down from demand; status flows back to everyone.

  1. Order in: A DTC shopper checks out on Shopify; a retail shopper's order arrives as an EDI 850 to Rithum. Both land in TechOMS.
  2. Orchestrate: TechOMS applies order rules, validates inventory, and routes the order to the correct warehouse/fulfillment channel.
  3. Fulfill: TechOMS sends a fulfillment order to Logiwa, which allocates stock, picks, packs and ships via a Carrier.
  4. Confirm: Logiwa returns the ship confirmation + tracking to TechOMS, which writes fulfillment/tracking back to Shopify and sends an 856 ASN (and 810 invoice) through Rithum to the retailer.
  5. Sync inventory: Logiwa on-hand changes flow up to TechOMS, which republishes available-to-sell to Shopify and Rithum to prevent overselling.

Responsibilities of Each System

System Primary Role Owns / Responsible For Does NOT Do
Shopify DTC Sales Channel Customer-facing storefront, DTC order capture, payment authorization/capture, customer notifications, DTC product catalog. Does not manage warehouse operations or retailer EDI. Relies on TechOMS for fulfillment + tracking writeback.
Rithum Retailer / Channel & EDI Hub Marketplace & dropship retailer connections, EDI translation (846/850/855/856/870/810), order normalization, channel inventory feeds, retailer compliance (SLA, labels, ASN timing). Does not pick/pack/ship and does not hold physical inventory. Routes orders to TechOMS and relays status back to retailers.
TechOMS Order Management / Orchestration Central order hub for all channels, order rules engine, dynamic fulfillment routing, inventory aggregation across channels, WMS sync (orders down, ship confirm up), recurring import/export jobs, IntegratedFlow connectors. Does not perform physical warehouse work; delegates fulfillment to Logiwa and channel comms to Shopify/Rithum.
Logiwa Warehouse Management (WMS) Physical inventory (system of record for on-hand), receiving & directed putaway, allocation/reservation, picking, packing, carrier label generation & shipping, returns receiving, cycle counts, multi-client billing. Does not talk to sales channels directly; receives fulfillment orders from TechOMS and returns ship confirmations to it.
Carriers Transportation Rating, label generation, package transport, tracking events, proof of delivery. No order/inventory logic. Integrated through Logiwa (and/or a carrier rate-shop service) for labels and tracking.

Integration Touchpoints

Link Direction Method Data Exchanged Frequency
Shopify -> TechOMS Inbound orders / Outbound fulfillment API (native connector / IntegratedFlow), webhooks + polling New DTC orders, customer & address, line items, payment status; back: tracking, fulfillment status, inventory levels Near real-time (webhook) + recurring poll (e.g. every 10-15 min)
Rithum -> TechOMS Inbound retailer orders / Outbound ship + invoice EDI (846/850/855/856/870/810), Dsco API, or flat-file/SFTP via Data Mapper Retailer POs (850), acknowledgments (855), ship notices (856), invoices (810), inventory (846), cancellations (870) Scheduled batches (e.g. every 15-30 min) per retailer SLA
TechOMS -> Logiwa Outbound fulfillment orders API / IntegratedFlow connector (Fulfillment Channel) Fulfillment orders, ship-to, items/qty, carrier/service, ship-by date, client/warehouse Recurring job (e.g. every 5-15 min) or near real-time
Logiwa -> TechOMS Inbound ship confirmations + inventory API / IntegratedFlow connector Ship confirmations (tracking, carrier, weight, ship date), short-ships, on-hand inventory snapshots Near real-time on ship; inventory on schedule (e.g. every 15-30 min)
Logiwa -> Carriers Label + tracking Carrier API / rate-shop integration Package dimensions/weight, service level; back: label (PDF/ZPL), tracking number Real-time at pack/ship

Common Synchronization Points

Sync Point Flow Why It Matters
Inventory availability Logiwa (on-hand) -> TechOMS (ATS) -> Rithum/Shopify (channel feed) Prevents overselling. Logiwa is the on-hand source of truth; TechOMS computes available-to-sell and pushes to channels.
Order capture Shopify/Rithum -> TechOMS All orders converge in TechOMS so rules, routing, and a single status model apply consistently.
Fulfillment dispatch TechOMS -> Logiwa Released/approved orders become warehouse fulfillment orders for picking.
Ship confirmation Logiwa -> TechOMS -> Shopify (fulfillment) & Rithum (856 ASN) Tracking must flow back same-day for many retailers; drives customer notifications and invoicing.
Receiving / inbound PO in TechOMS -> Logiwa receiving -> on-hand increment -> inventory feed to channels New stock becomes sellable only after received and synced upward.
Returns Return created (Shopify/Rithum/TechOMS) -> Logiwa receiving/inspection -> inventory adjust -> status/refund back Restocks sellable units and closes the loop with the customer/retailer.
⚠️
Most incidents happen at sync points

When something looks wrong, first ask: which sync point failed and which direction? An order missing from the warehouse is a TechOMS→Logiwa issue; overselling is an inventory feed issue; a retailer chargeback for a late ASN is a Logiwa→TechOMS→Rithum issue.
